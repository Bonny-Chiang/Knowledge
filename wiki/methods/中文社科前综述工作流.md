# 中文社科前综述工作流

## Purpose

本方法页用于组织中文社科研究的前综述流程，把“检索、筛选、入库、模块化综述、NotebookLM 提问、结果回收”拆成可审计的阶段。

## Workflow

```text
研究问题校准 -> 关键词校准 -> CNKI/Scholar 检索 -> 候选清单 -> 人工审核 -> Zotero 入库与 PDF 附件 -> 模块化综述框架 -> NotebookLM 提示词 -> 可见提问 -> 原始回答回收 -> 定向整理
```

## Control Points

- 候选文献清单生成后必须停止，等待人工审核。
- 未经确认，不进入 Zotero、全文下载或 Obsidian 汇总。
- Zotero 入库必须同时满足条目进入目标 collection 且 PDF 已附加。
- NotebookLM 适合做 grounded 阅读、比较和模块化整合，不应替代人工判断。
- NotebookLM 回答回收时必须先保存原始版，再按用户要求整理。

## Durable Insight

前综述不是“让 AI 直接写综述”，而是把证据准备和写作准备分开：先建立可核验的文献集合，再把文献组织成问题导向的模块，最后才进入局部写作或整理。
