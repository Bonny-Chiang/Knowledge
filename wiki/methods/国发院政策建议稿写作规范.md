# 国发院政策建议稿写作规范

## Purpose

本方法页用于指导国发院内参/政策建议类稿件写作，尤其适合将调研、案例、数据和政策问题整理成可报送的决策参考文本。

## Selection Rules

- 选题要重大，具有现实政策意义。
- 注重时效，回应当前热点问题。
- 一事一议，避免一篇稿件承载过多议题。
- 可以围绕相关部门特定选题约稿展开。

## Content Rules

- 遵循科研诚信和学术规范。
- 语言要凝练。
- 数据要详实。
- 对策要务实。
- 每个观点都应有数据或案例支撑。
- 避免泛泛而谈和信息陈旧。
- 关键表述、数据、案例必须准确无误。
- 数据和案例应以脚注注明出处，包括网络链接、参考文献或具体资料。
- 如果无法获取或未使用最新数据，应以脚注说明原因。

## Length

- 常规政策建议：全文约 3000-4000 字，正文前附约 500 字摘要。
- 深度研究类报告：全文约 6000-8000 字，正文前附约 500 字摘要。

## Recommended Structure

```text
标题
作者信息
摘要
帽段/背景
一、基本情况
二、存在的问题
三、对策建议
第一作者联系方式
```

另一种样刊结构是：

```text
摘要
背景/调研情况
一、主要问题
二、深层次原因分析
三、对策建议
```

## Style Pattern

正文常用“一是、二是、三是”的并列论述方式，将事实、问题和建议压缩成清晰的政策表达。

## Sample Report Patterns

[[../sources/关于我国蚕桑产业发展情况的报告]] 和 [[../sources/关于高校建设农业产业研究院有关情况的报告]] 可以作为本规范的样本库，而不只是作为农业主题资料。

它们共同展示了政策报告常用的“基本情况 -> 主要问题 -> 政策方向”结构：

- 先用数据和事实建立问题的现实基础。
- 再把行业或平台运行中的分散现象压缩成少数关键问题。
- 最后将问题翻译成可执行的政策动作。

这一路径可用于将 [[../projects/政务无人机运行数据标准化政策建议]] 从一般建议稿升级为更符合内参要求的稿件：先证明政务无人机应用已经形成规模，再证明数据标准缺失正在成为治理瓶颈，最后提出标准体系建设方案。

## Using Web Evidence Packs

像 [[../sources/低空经济一网统飞资料汇总（网页版）]] 这类网页资料包，适合充当“基本情况”和“政策依据”的脚注库，但不应把不同证据强度的网页混用。

更稳妥的用法是：

- 优先使用 `gov.cn` 部门文件、政策解读和发布会口径支撑制度背景、时间线和官方表述。
- 将省市平台案例用于说明实践进展、平台架构和运行指标，并在脚注中明确城市、时间和原始网页。
- 将行业观察文章用于概念解释、问题意识和对比表述，但避免让它单独承担核心政策判断。
- 下笔前先把材料整理成“国家政策依据 / 地方案例 / 行业观察”三栏，再写摘要、帽段和正文。

这种分层使用证据的方法，尤其适合需要同时处理政策文件、案例材料和网络来源的低空经济类选题。

## Submission and Review

- 投稿邮箱：`nads07@ruc.edu.cn`
- 约稿和报送联系：国发院成果转化中心工作人员。
- 审核流程：国发院内参编辑部审核 -> 学校政策研究室审核 -> 分管智库副校长审核 -> 学校党委书记审核。
- 工作日期间，录用情况通常每 2 周以邮件形式通知。
- 上级单位采用反馈时间一般为 2-3 个月，假期顺延。

## Reusable Checklist

- [ ] 题目是否重大、及时、一事一议？
- [ ] 摘要是否提炼出有观点性的关键判断？
- [ ] 帽段是否言简意赅交代背景和问题？
- [ ] 每个核心观点是否有数据或案例支撑？
- [ ] 关键数据、案例和表述是否有出处？
- [ ] 问题诊断是否避免泛泛而谈？
- [ ] 对策是否具体、务实、可执行？
- [ ] 字数和格式是否符合要求？
- [ ] 是否附第一作者联系方式？
