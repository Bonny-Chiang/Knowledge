# 政务无人机运行数据标准化框架

## Purpose

该框架用于把政务无人机从“统一飞行调度”升级为“统一数据治理”，使飞行成果能够被跨部门复用、质量评价、安全管理和绩效评估。

## Why It Is Urgent

新收集的网页资料显示，地方“一网统飞”平台已经进入规模化运行阶段，而国家低空标准体系也在加快完善。当前的制度空缺不再是有没有平台，而是平台运行数据能否被稳定采集、纳入目录、评价质量、安全共享并沉淀为公共数据资源。

换句话说，低空标准体系正在变多，但直接面向政务无人机运行数据复用的标准还需要进一步细化。

## Six Standards

### 1. 数据采集标准

统一任务编码、时空基准和采集参数。

关键要素包括飞行高度、航线间距、拍摄角度、影像分辨率、坐标系统、时间戳、设备参数和气象记录。

### 2. 数据目录标准

将政务无人机运行数据纳入公共数据资源目录。

数据类型包括设备、人员、任务、航线、影像、视频、三维模型、AI 识别和事件处置结果。

### 3. 数据质量标准

建立可评价、可复核、可追溯的指标体系。

核心指标包括完整性、准确性、清晰度、时效性、一致性、可追溯性和可复核性。

### 4. 共享接口标准

统一跨部门、跨层级、跨平台的数据调用规则。

重点包括元数据格式、API 接口、授权流程、成果反馈机制和调用记录。

### 5. 安全管理标准

建立分类分级、最小必要、按需授权、全程留痕和责任追溯机制。

适用于重点区域、基础设施、公共安全事件、人群活动和高清地理空间信息等敏感数据。

### 6. 绩效评价标准

从“飞行数量”评价转向“治理成效”评价。

重点指标包括任务响应时间、跨部门调用次数、数据复用次数、重复飞行减少比例、事件闭环率、单位任务成本、数据质量达标率和敏感数据调用留痕率。

## Operating Logic

```text
采集标准 -> 目录标准 -> 质量标准 -> 共享接口 -> 安全管理 -> 绩效评价
```

## Spatial-Model Extension

包河区材料说明，政务无人机运行数据标准化还应覆盖空间识别和节点规划模型：

- 任务需求指标口径：人口、夜光、POI、道路、建设用地、生态占比和水系距离等代理变量如何定义。
- 空间单元标准：1km 网格、500m 网格、街道或社区单元如何选择和比较。
- 模型过程记录：标准化方式、权重规则、相关分析、随机森林参数和变量筛选记录。
- 候选点约束：禁飞区、净空、建筑高度、用地权属、电力接入和安全缓冲。
- 结果复核标准：候选网格不得直接等同于正式点位，必须经过实地和部门任务数据核验。

## Implementation Sequence

结合公共数据治理类政策文件，较稳妥的落地顺序是：

1. 先统一任务编码、时空基准和采集规则，避免源头数据不可比。
2. 再把运行数据纳入目录和共享清单，明确哪些数据应归集、哪些可以共享。
3. 同步建立质量校核、调用留痕和分类分级规则。
4. 最后用跨部门调用、重复飞行减少比例、事件闭环率和单位任务成本来检验平台建设成效。
