# 包河区政务无人机巡查需求空间识别

## Project Summary

该项目以合肥市包河区为试点，将政务无人机巡查需求转译为空间识别问题，使用公开空间数据构建 1km 网格指标表，并形成巡查需求指数、起降点适宜性图和变量重要性结果。

## Core Questions

- 政务无人机潜在巡查需求在城市空间中如何分布？
- 哪些区域更适合作为起降点、充电点和指挥节点候选片区？
- 在缺少真实政务飞行任务数据时，哪些公开空间指标可以作为代理变量？

## Current Findings

- 高需求区域主要集中在夜间活动强、人口较密集、POI 与道路设施较集中的城市化片区。
- 需求指数范围为 0.2206-0.6208。
- 夜间灯光、人口密度、POI 密度和道路密度是关键解释变量。
- 起降点适宜性前五位网格为 BH0253、BH0324、BH0286、BH0192、BH0335。
- 需求图回答“哪里更需要巡查”，适宜性图进一步回答“哪里更适合布设节点”。

## Planning Implications

- 政务无人机起降点、充电点和指挥节点应纳入公共服务设施、城市安全设施和低空基础设施统筹布局。
- 适宜性网格只能作为后续精细化核查对象，不能直接等同于最终建设点位。
- 真实规划阶段需要补充禁飞区、净空条件、建筑高度、用地权属、电力接入和真实部门任务数据。

## Reproducibility Requirements

后续若继续推进该项目，应将道路密度、POI 密度、水系距离、人口覆盖、候选点筛选等步骤整理为 [[../methods/QGIS Processing Toolbox ETL 工作流]] 或脚本化流程。

最低限度应保存：

- 输入数据版本和原始来源。
- 研究区边界、网格尺度和 CRS。
- QGIS Processing Toolbox 工具参数。
- 字段计算表达式和筛选条件。
- 中间 temporary layer 与最终 GeoPackage 输出的区别。
- 最终指标表、地图版式和生成日期。

## Link to Policy Writing

该项目为 [[政务无人机运行数据标准化政策建议]] 提供了更强的实证支撑：它不只说明政务无人机需要标准化，还展示了运行数据、空间指标和候选节点评价如何转化为可复核的政策证据。
