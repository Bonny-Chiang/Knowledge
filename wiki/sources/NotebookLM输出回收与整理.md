# NotebookLM输出回收与整理

## Source

- Raw file: `raw/github/research-review-03-notebooklm-recovery.md`
- Source type: AI workflow / skill document
- Scope: NotebookLM 多轮回答经人工审阅后，回收、保留原始版、按要求整理并归档到 Obsidian。

## Key Claims

- 回收必须由用户明确触发，不主动抓取 NotebookLM 内容。
- 整理前先保存尽量完整的原始版本。
- 回收策略需明确标记为 `latest`、`user-specified` 或 `multi-turn-merge`。
- 用户未给整理要求时，只做最小加工，不默认总结或重写。
- 多轮整合不能冒充单轮原文，冲突必须标出。
