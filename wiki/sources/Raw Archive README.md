# Raw Archive README

## Source

- Raw file: `raw/README.md`
- Document type: repository README
- Scope: archive rules for the `raw/` layer

## Summary

This README defines `raw/` as the source-of-truth archive for the knowledge base. It emphasizes preserving original materials, keeping a stable folder structure for predictable retrieval, and recording provenance through sidecar metadata when needed.

## Key Points

- `raw/` stores original materials such as papers, webpages, repositories, images, video, datasets, audio, and books.
- Original source files should not be manually edited after capture.
- Sidecar notes should preserve provenance details like URL, author, access date, license, and capture method.
- Stable subfolders such as `inbox/`, `papers/`, `web/`, `github/`, `images/`, `video/`, `subtitles/`, `datasets/`, `audio/`, and `books/` make archive retrieval predictable.
