## Summary

这份视频稿面向 GitHub 初学者，用非专业术语解释 GitHub 的核心用途：把本地代码放到云端、建立版本检查点、支持协作、利用开源代码，并在真实项目中通过分支、提交、拉取请求和 `.gitignore` 降低开发风险。

## Key Ideas

- GitHub 的基础价值是把本地项目文件夹变成云端 repository，并提供可回退的版本历史。
- Commit 可以理解为项目检查点；push 是把本地变化送到云端；pull 是把云端更新同步回本地。
- Branch 用于在不破坏 `main` 的情况下开发新功能。
- Pull request 用于把分支中的变化合并回 `main`，并在合并前进行检查、讨论和审阅。
- Fork 用于复制别人的公开仓库到自己的账号下，便于学习、修改或贡献。
- `.gitignore` 用于阻止敏感文件、日志、环境变量和不必要文件进入 GitHub。
- SSH key 用于让本地电脑和 GitHub 建立可信连接。

## Relevance to Knowledge Base

对当前个人知识库而言，这个视频的价值不只是“学习 GitHub”，而是补全 Finder + Obsidian + GitHub 知识库的版本控制心智模型：不要把 GitHub 仅理解为上传工具，而要理解为检查点、协作边界、恢复机制和远程备份。
