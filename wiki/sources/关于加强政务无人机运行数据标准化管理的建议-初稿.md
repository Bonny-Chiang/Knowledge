# 关于加强政务无人机运行数据标准化管理的建议-初稿

## Source

- Raw file: `raw/papers/关于加强政务无人机运行数据标准化管理的建议-初稿.docx`
- Document type: 政策建议初稿
- Authors: 杨琳、姜鹏辉
- Institution: 中国人民大学公共管理学院

## Core Claim

随着低空经济加快发展，政务无人机正在从部门自用、单点试用转向平台化、规模化、场景化应用。下一阶段的关键问题不只是“把无人机飞起来”，而是把飞行任务、航线轨迹、视频图像、三维模型、AI 识别结果、事件处置结果等运行数据规范采集、统一归集、安全共享和持续复用。

## Problem Diagnosis

文档提出六类问题：

1. 平台建设快于标准建设，可能造成重复建设和低水平接入。
2. 数据采集标准不一，影响横向比较、历史比对、模型训练和效果评估。
3. 数据质量评价不足，容易形成“飞了很多、数据不好用”。
4. 共享接口不统一，制约跨部门“一飞多用”。
5. 敏感数据安全边界不清，存在泄露和滥用风险。
6. 绩效评估标准缺位，难以判断平台建设成效。

## Proposed Standard System

文档建议建立覆盖六类标准的政务无人机运行数据标准体系：

| 标准类别 | 重点内容 | 主要指标或要求 |
| --- | --- | --- |
| 数据采集标准 | 统一不同场景下飞行、拍摄、定位、编码等采集规则 | 飞行高度、拍摄角度、分辨率、坐标系统、时间戳、任务编码 |
| 数据目录标准 | 明确运行数据分类和公共数据资源归集范围 | 设备、人员、任务、航线、影像、视频、三维模型、AI 识别、处置结果 |
| 数据质量标准 | 建立可评价、可复核、可追溯的质量指标 | 完整性、准确性、清晰度、时效性、一致性、可追溯性 |
| 共享接口标准 | 统一跨部门、跨层级、跨平台调用规则 | 元数据格式、API 接口、授权流程、反馈机制、调用记录 |
| 安全管理标准 | 明确敏感数据分类分级和责任链条 | 脱敏处理、访问权限、留痕审计、重点区域保护、违规追责 |
| 绩效评价标准 | 从“飞行数量”转向“治理成效”评价 | 响应时间、复用次数、重复飞行减少比例、事件闭环率、单位任务成本 |
