# 前综述NotebookLM提示词与执行

## Source

- Raw file: `raw/github/research-review-02-notebooklm-prompts-execution.md`
- Source type: AI workflow / skill document
- Scope: 文献筛选完成后，搭建模块化综述框架、生成 NotebookLM 提示词，并在用户准备好 notebook 后可见执行提问。

## Key Claims

- 综述框架必须围绕研究问题组织，而不是按作者或文献列表堆砌。
- NotebookLM prompt 应服务论文写作，强调共识、差异、争议和研究空白。
- 在用户创建 notebook、上传资料并明确授权前，不得自动进入提问执行。
- 提示词应要求 APA 风格文中引用和参考文献列表。
- 若自动上传 PDF，必须回查来源区文件名和来源数量，不能只以文件选择器触发为成功。
